# PromptCore — Version V2

> PromptCore 의 버전 이력과 병합 출처를 기록한다. 법적 라이선스 표기는 루트 `ATTRIBUTION.md` 가 정본이며, 이 파일은 *무엇을 어디서 병합했는지*의 추적 기록이다.

## V2 — 2026-06-14 (초기 정본)

### 변경 요지

- PromptCore 의 첫 정본을 `core/promptcore.md` 에 작성했다.
- 두 출처를 inspired-by 병합했다. 전문 복제 없이 패턴·구조만 추상화했다.
  - `sgup/ai` 의 Fable5.md operating profile → §1 운영 정체성, §2~§5 절차 규율의 행동 프로필.
  - `value-for-fable` 의 8섹션 → §6 출력구조, §7 가성비 운영, §8 drift/passive 운영.
- 정찰 실측 보정을 §0 에 명시했다. capability(전이 불가) vs procedure(전이 가능) 구분을 정체성 전제로 고정했다.

### 채택·배제 근거

| 항목 | 결정 | 근거 |
|------|------|------|
| Fable5.md operating profile | 채택(패턴만) | 전이 가능한 행동 규율(proactive·grounded·완결성). 전문은 복제하지 않고 §1·§2~§5 로 추상화 |
| value-for-fable 출력구조 8섹션 | 채택(구조만) | 출력 일관성·검증성에 직접 기여. §6 으로 통합 |
| value-for-fable 압축 강제 규칙 | **배제** | value-for-fable v2 교훈상 품질 부채. 산출물 분량을 강제로 줄이는 규칙은 PromptCore 에 넣지 않음(REQUIREMENTS FL3). §6 의 "압축"은 컨텍스트 상태 스냅샷에 한정 |
| 모델 capability 전이 주장 | **배제** | 정찰 실측상 불가(capability 비전이). §0 에 정직하게 명시하고, 효과의 정량 주장은 `bench/` 로 위임(FL9) |

### 검증 상태

- license_gate 자가 점검 대상: 전문 미포함·카피 지시 부재·압축 강제 규칙 부재. 통과 여부는 `bash gates/license_gate.sh` 종료코드로 판정한다(자가보고 아님).
- 정량 효과 주장: 본 버전은 정량 주장을 포함하지 않는다. 효과 측정은 `bench/` 로 위임한다.

## 버전 승급 규칙

- 새 버전은 `core/versions/v<N>.md` 를 추가하고 이 파일 형식(변경 요지·채택 근거·출처 매핑·검증 상태)을 따른다.
- 이전 버전 파일은 파괴적으로 덮어쓰지 않는다.
- `core/promptcore.md` 상단 버전 표기를 새 버전으로 갱신하되, 변경된 섹션만 수정하고 나머지는 보존한다.
